Design direction showcase

Six chrome layouts for the same full-bleed black stage (illustrative gradient stands in for Three.js). Use this to compare hierarchy before locking a direction in the UX spec.

High-contrast focus ring (keyboard tab to HUD)
Direction 1 — Bottom bar
Live v0.1

Pro-tool feel: one stable strip; status and primary action together. Strong default for desktop-first.

Direction 2 — Top strip
visualize-music
Ready

Title + state read at a glance; maximum void below. Good if you want a subtle “app” label without crowding the center.

Direction 3 — Corner stack
Mic
Idle

Floating card; the stage feels uninterrupted. Slightly more “object” in the void—watch overlap on very small viewports.

Direction 4 — Center pill

Minimum chrome: one control cluster. Mysterious first-land; pair with plain error surfaces elsewhere so failures never vanish.

Direction 5 — Left rail

Theatrical “wing” of controls; the stage reads as a proscenium slice. Unusual; test tab order and narrow widths.

Direction 6 — Top status + bottom CTA
Input Listening
Help

Separates read-only state (top) from action (bottom). Clear when you add a secondary line (help, privacy) without duplicating the primary CTA row.
