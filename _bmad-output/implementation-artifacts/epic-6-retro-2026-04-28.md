- **Clear layering:** detector ‚Üí map ‚Üí smooth ‚Üí frame ‚Üí graph events ‚Üí scene spawns stayed separated; no rAF ‚Üí React thrash.
- **Testable core:** synthetic spectra and unit tests for HPS, harmonic prune, pitch classes, and graph chord shape.

## Challenges (systems, not blame)

- **Acoustic reality ‚âÝ unit tests:** harmonics and noise create extra pitch classes until thresholds and pruning match the room.
- **Two control knobs in tension:** poly sensitivity vs false positives ‚Äî still tunable with `peakFloor`, `suppression`, `POLY_EVENT_MIN_CONF`, `MONO_PITCH_CONF_SPAWN`.
- **Planning doc lag:** `epics.md` does not yet have a dedicated ‚ÄúEpic 6‚Äù narrative; stories lived in `implementation-artifacts` ‚Äî fine for delivery, easy to lose context for newcomers.

---

## Lessons learned

1. **Normalize before HPS** (linear spectrum max scale) so peak floors behave across mics and gain.
2. **Prune harmonics in Hz** before mapping to pitch class ‚Äî class collisions (f vs 2f) were the main ‚Äúghost chord‚Äù source.
3. **Monophonic hint** is a good override for *spawns* when confidence is high, without deleting poly from the whole pipeline.
4. **Chords and monophony need different product tests:** automated tests for math; manual listening for ‚Äúfeels right.‚Äù
